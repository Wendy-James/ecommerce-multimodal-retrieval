# Boundary Statement

This public repository is a sanitized reproduction of the workflow, not the original internship code or data.

It uses pseudo/anonymized ecommerce examples and does not publish merchant data, product images, platform labels, user data, internal tables, or production code.

The resume internship description only refers to offline evaluation, sample analysis, threshold-metric tables, hard-negative review, and badcase work that I can explain. It does not claim ownership of a full production search/recommendation system.

What I can discuss in an interview:

- field design for title-image pairs;
- CLIP-style dual-encoder retrieval logic;
- hard negative construction and false-negative risk;
- Recall@10 and NDCG@10 evaluation;
- SKU conflict, OCR noise, title generalization, and same-category different-style badcases;
- why this supports strategy evaluation rather than claiming online A/B ownership.
